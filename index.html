OCAP Forecast Product Eligibility & Status Assignment

Truckee & Carson River Basins • Forecast Relevance (Definition → Eligibility → Comparative Preference)
Interactive Placemat
Conservative by design: if you cannot verify a criterion, leave it unchecked.
1. Definition i
What this gate decides: Is the “product” a forecast, or just a measurement/diagnostic?

What to ask for:
  • Show an example output. Does it include a forecast hydrograph or forecasted volume?
  • Does it predict future flow/inflow, or only current conditions (e.g., SWE)?
  • What is the forecast horizon (days, months, seasonal period)?
Common examples: SWE maps = diagnostic. Inflow ensemble hydrograph = forecast.
○
A forecast must predict future streamflow or reservoir inflow as a time series and/or volume over a defined horizon.

Stop if any unchecked

If any item above is unchecked, treat the product as Observational/Diagnostic (not a forecast). It may inform forecasts but cannot set OCAP targets directly.

2. Operational Eligibility i
What this gate decides: Even if it is a forecast, is it operationally eligible for OCAP consideration?

What to ask for:
  • Can it be turned into an OCAP input without expert judgment?
  • Does it cover a horizon relevant to storage targets?
  • Is it produced by an operational steward with continuity?
  • Is the method documented so it can be audited?
  • Does a hindcast/reforecast record exist so performance can actually be evaluated?
 ○
This gate determines whether the forecast can enter the pool of products considered for OCAP use. It screens practicality, continuity, and auditability.

Stop if any unchecked

If any item above is unchecked, the forecast is operationally ineligible for OCAP target setting, even if it may still be useful for research or situational awareness.

3. Skill / Comparative Preference i
What this gate decides: Among operationally eligible forecasts, how much value does this source add for the basin and decision horizon required by OCAP?

What to ask for:
  • Hindcast/verification report for Truckee/Carson
  • Metrics using accepted forecast verification methods (e.g., bias, RMSE/MAE, CRPS or other probabilistic scores for ensembles)
  • Comparison to baseline (climatology + persistence)
  • Evidence for the same horizon used for OCAP targets
  • Evidence that the source adds information for uncertainty or important hydrologic regimes
Forecast verification should follow well-established methods in the forecast verification literature to ensure the framework remains defensible and durable over time.
○
A forecast may only be considered if evidence demonstrates predictive value beyond baseline methods (e.g., climatology + persistence), or contributes useful information for representing forecast uncertainty or hydrologic regime behavior.

Status if not fully checked

If Gate 3 is incomplete, the forecast may still remain eligible, but its role should generally be limited to monitoring / parallel testing or, where justified, supplemental use until stronger comparative evidence exists.

  • If multiple forecasts are used, define the combination/prioritization method in advance to prevent forecast-shopping.
  • Status assignment should be tied to the intended OCAP use case and documented review procedures.

Definition

Operational Forecast: A quantitative prediction of future streamflow or reservoir inflow, expressed as a time series or volumetric quantity over a defined forecast horizon, suitable for direct use in setting storage objectives and related operational calculations.

Observational/Diagnostic Products: Describe existing/antecedent conditions (e.g., SWE, soil moisture, antecedent precipitation, etc.). These are typically inputs that inform eligible forecasts, but are not forecasts themselves and shall not directly set OCAP targets unless they independently meet the forecast definition.

Operational Eligibility

  • Direct output usable without discretionary interpretation
  • OCAP-relevant horizon for storage/diversion decisions
  • Operational stability and continuity (not experimental)
  • Transparency sufficient for independent audit
  • Integration pathway within existing OCAP/TROA frameworks
  • Hindcasts using only data available at issuance

Comparative Preference

  • Basin-specific evaluation for Truckee/Carson
  • Horizon-matched skill for OCAP decision horizon
  • Metrics (bias/error and probabilistic where applicable)
  • Baseline comparison against climatology + persistence
  • Status assignment based on evidence for monitoring, supplemental, or primary/weighted use
